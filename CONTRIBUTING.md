# Contributing to scd-zig

Thanks for your interest. This is a focused, single-purpose Zig primitive.

**Bar:** every change keeps `zig build test` green. Performance claims are
paired with a benchmark; correctness claims with a test. No claim ships above
the evidence: sketch → compiled → unit-tested → integration-tested → audited → benchmarked.

**Getting started**
1. `zig build test` — all tests must pass before and after your change.
2. Keep changes scoped and minimal; match the surrounding style.
3. Open a PR with a clear description and the evidence level your change reaches.

Good first issues are labeled `good first issue`. Questions welcome via issues.
